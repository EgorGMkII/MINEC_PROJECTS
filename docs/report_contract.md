# Report contract

The report is generated by the final prompt in `summary_rag.py` using the active
base system prompt.

Developer mode uses `rag_support.DEFAULT_SYSTEM_PROMPT`, which asks for an
internal self-check style:

- overall project assessment;
- key errors;
- why each issue matters;
- what to check or clarify;
- how to fix it;
- additional risks;
- final conclusion.

Public mode uses `old_prompt.OLD_SYSTEM_PROMPT`, which asks for a more official
draft conclusion style. Do not merge these modes accidentally.

The final answer also includes the UFAS precheck block produced before the main
RAG answer.

When changing checking logic or report format, explicitly assess:

- risks;
- side effects;
- possible regressions;
- whether the answer still prioritizes central substantive defects over generic
  KPI/finance/formal remarks.

Запрещено:
- выдавать предположения как факт;
- выдумывать отсутствующие сведения;
- удалять UFAS precheck без отдельного решения;
- подменять конкретный разбор проекта набором типовых замечаний.
